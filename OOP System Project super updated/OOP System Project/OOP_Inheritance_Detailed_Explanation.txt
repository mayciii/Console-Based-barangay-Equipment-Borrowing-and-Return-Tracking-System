OOP Inheritance Detailed Explanation
====================================

This document provides a comprehensive explanation of the inheritance part in the OOP System Project, focusing on the Account, Admin, and User classes. It covers key OOP concepts like inheritance, instance variables, instantiation, polymorphism, and abstraction with line-by-line code breakdowns.

1. **Inheritance Overview**
   Inheritance is a fundamental OOP concept where a class (subclass or child class) inherits properties and behaviors from another class (superclass or parent class). This promotes code reuse and establishes an "is-a" relationship.

   In this system:
   - Account is the base class (parent).
   - Admin directly inherits from Account.
   - Borrower is an abstract class that inherits from Account.
   - User inherits from Borrower (indirectly from Account).

2. **Key OOP Concepts Explained**

   a. **Instance Variables**
      - These are variables declared in a class that hold data unique to each object instance.
      - They are initialized during object creation (instantiation).
      - Example: In Account.java, 'username' and 'password' are instance variables.

   b. **Instantiation**
      - The process of creating an object from a class using the 'new' keyword.
      - This calls the class constructor to initialize the object.
      - Example: 'new Account("user", "pass")' creates an Account object and sets its instance variables.

   c. **Polymorphism**
      - Allows methods to behave differently based on the object calling them.
      - Types: Method Overloading (compile-time) and Method Overriding (runtime).

   d. **Abstraction**
      - Hiding complex implementation details and showing only essential features.
      - Achieved through abstract classes and interfaces.

      - This is the root class in the inheritance tree.

        - public class Account {  // Declares the Account class.
        - private String username;  // Instance variable for username (private for encapsulation).
        - private String password;  // Instance variable for password (private for encapsulation).
        - public Account(String username, String password) {  // Constructor for instantiation.
          - this.username = username;  // Sets the username instance variable.
          - this.password = password;  // Sets the password instance variable.
        - }
        - // Getter/setter methods provide controlled access to private variables.
        - public boolean login(String username, String password) {  // Method using instance variables.
          - return this.username.equals(username) && this.password.equals(password);  // Compares instance variables.
        - }

      - Inherits from Account but cannot be instantiated directly.
      - Demonstrates abstraction by defining abstract methods.

        - public abstract class Borrower extends Account {  // Inherits from Account, marked abstract.

        - public abstract void returnItem(...);  // Abstract method: must be implemented by subclasses.

4. **Instantiation Examples**
   - Account acc = new Account("user", "pass");  // Creates Account, initializes username/password.
   - Admin admin = new Admin("admin", "pass");  // Creates Admin, initializes inherited + equipmentList/logs.
   - User user = new User("user", "pass");  // Creates User, initializes inherited + borrowedItems.

5. **Polymorphism in Action**
   - **Overloading**: Admin's addEquipment methods allow flexible calls.
   - **Overriding**: User's borrowItem and returnItem provide specific implementations.

6. **Abstraction Benefits**
   - Borrower abstracts borrowing behavior, allowing User to focus on implementation details.
   - Promotes modularity: Code is organized, maintainable, and extensible.

This inheritance structure enables Admin and User to share Account's authentication while specializing their behaviors, demonstrating effective OOP design.
